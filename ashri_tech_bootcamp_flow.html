Students build the player identity system. They type their name — and it appears on screen in the game. Variables stop being abstract boxes and become real things they can see.

What they build: Name Tag Simulator → Player enters name, pilot rank, and ship name. Game displays animated intro card. First moment of "MY code is on screen."
# Galaxy Defender — Day 2 import pygame player_name = input("Enter your name: ") ship_name = input("Name your ship: ") score = 0 health = 100 # Display on screen screen.blit(font.render(player_name, True, WHITE), (10, 10))
Maths & Damage
The Game Learns
to Calculate
The health bar appears. The score counter ticks. Students see maths happening in real time — and suddenly arithmetic isn't boring, it's the reason their ship survives.
💊
Damage System: health = health - 10 shows visually as the health bar shrinking. Operations become cause and effect.
🔄
Typecasting explained as: "Translating between human language and machine language." int(), str(), float() — necessary for the score display to work.
# Health & Score System health = 100 damage = 10 score = 0 kill_reward = 50 # When hit health = health - damage # When enemy killed score = score + kill_reward # Display label = "SCORE: " + str(score)

Decision Making
Teaching the Game
to Think
The game starts making decisions. Health drops to zero? GAME OVER screen fires. Player enters a danger zone? Damage triggers. The if/elif/else becomes the brain of the game.
🧠
Rules Metaphor: "Conditions are the rules of your game world." if health <= 0: means "if the player dies, do this."
🏆
Quiz Battle Activity: Answer correctly → attack enemy on screen. Wrong → lose health. Concepts and game mechanics are now the same thing.
# Game decision system if health <= 0: game_state = "GAME_OVER" show_game_over() elif health < 30: # Flash health bar red health_color = RED else: health_color = GREEN
The Heartbeat
Everything Starts
Moving
The most important day. The game loop is introduced — and suddenly EVERYTHING moves. Stars scroll. The player ship responds. The game feels alive for the first time.
❤️
The Heartbeat Metaphor: "A while loop is the heartbeat of the game. Without it — everything freezes. With it — everything moves forever."
🔁
Demo contrast: First show game WITHOUT the loop — students see a frozen frame. Then add the loop — suddenly things move. That contrast is unforgettable.
# The Game Loop — the heartbeat running = True while running: # 1. Handle inputs for event in pygame.event.get(): if event.type == QUIT: running = False # 2. Move player player_x += speed # 3. Draw everything draw_player() pygame.display.update()

06
Reusable Powers
The Ship Gets
Special Abilities
Functions are introduced as "special abilities" the game can call. shoot_bullet(), spawn_enemy(), draw_player() — students realise they're building a library of powers for their game world.
⚡
Superpower Metaphor: "Every time your ship presses SPACE, it calls shoot_bullet(). A function is a named superpower you can trigger anywhere."
🏭
Enemy Factory: A function that creates an enemy at a given position. Call it once and one appears. Call it 10 times — 10 enemies. The power of reuse made visual.
def shoot_bullet(x, y): bullet = {'x': x, 'y': y, 'speed': 8} return bullet def spawn_enemy(x, y): enemy = {'x': x, 'y': y, 'hp': 3} return enemy def draw_player(surface, x, y): pygame.draw.polygon(surface, GREEN, [(x, y), (x-20, y+40), (x+20, y+40)])
Teams & Armies
One Bullet Becomes
a Blizzard
Lists unlock the most exciting upgrade yet — the player can fire MULTIPLE bullets simultaneously. Students go from one bullet that disappears to a stream of projectiles flying across the screen.
👥
Team Metaphor: "A list is a team. Instead of one bullet — a whole squad. Instead of one coin — a bag full of coins. Lists let you manage MANY things at once."
🌧️
Asteroid Rain: Store 20 asteroids in a list. Loop through them all. Move them all. Remove them when off-screen. The list is the reason this is even possible.
# Bullet list bullets = [] # On SPACE press if keys[K_SPACE]: bullets.append(shoot_bullet(player_x, player_y)) # Move all bullets for bullet in bullets: bullet['y'] -= bullet['speed'] # Remove off-screen bullets = [b for b in bullets if b['y'] > 0]

Armies & Waves
Enemy Waves
Invade
Nested lists power the wave system. Rows of enemies move in formation. When one wave is cleared, the next spawns — harder, faster. The game now has proper progression and challenge.
🏫
Classroom Metaphor: "A nested list is like a classroom building. The building is the outer list. Each class is an inner list. Each student is an object inside."
🌊
Wave System: Enemies are stored as a list of lists. When enemies[] is empty, wave number goes up and a new formation spawns — building difficulty naturally.
# Enemy wave system wave_1 = [ spawn_enemy(100, 50), spawn_enemy(200, 50), spawn_enemy(300, 50), ] if len(enemies) == 0: wave_number += 1 enemies = create_wave(wave_number)
Chaos & Surprise
The Game Becomes
Unpredictable
Randomness makes the game feel truly alive. Enemy positions change every run. Loot drops in different spots. No two games are the same — and students discover their gamer tag using string slicing.
🎲
Chaos Metaphor: "Randomness = unexpected events. Without it, players learn your pattern in 10 minutes. With it, the game stays fresh forever."
🏷️
Gamer Tag Activity: name[:3] creates a 3-letter tag from their name. Students generate secret codes, call signs, usernames — string slicing suddenly has personality.
import random # Random enemy spawn enemy_x = random.randint(50, SCREEN_W - 50) enemy_y = random.randint(-100, -30) # Random loot drop if random.randint(1, 100) < 20: # 20% chance spawn_powerup(enemy_x, enemy_y) # Gamer tag from name gamer_tag = player_name[:3].upper() + "_ACE"
